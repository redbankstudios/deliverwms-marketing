# Deliver WMS — Marketing Site

Public marketing site for Deliver WMS, deployed at `deliverwms.com`. The authenticated app lives at `app.deliverwms.com` (separate Lovable project).

Built with [Astro](https://astro.build) + [Tailwind CSS](https://tailwindcss.com). Static output, deployed on Vercel.

## Development

```bash
npm install
npm run dev
```

Visit `http://localhost:4321`.

## Environment variables

Copy `.env.example` to `.env.local` and fill in the values:

- `PUBLIC_SUPABASE_URL` — app-side Supabase project URL (used for form submissions)
- `PUBLIC_SUPABASE_PUBLISHABLE_KEY` — Supabase anon key
- `PUBLIC_TURNSTILE_SITE_KEY` — Cloudflare Turnstile site key (Contact form anti-bot)
- `PUBLIC_APP_URL` — base URL for Log In / Sign Up CTAs (production: `https://app.deliverwms.com`)

## Deploy

Connected to Vercel via GitHub. Pushes to `main` auto-deploy to production. Pull requests get preview deploys.

## Structure

```
src/
  pages/         Astro pages, one per route
  components/    Shared components (mostly .astro, React for islands)
  layouts/       Page-level layouts (header, footer, head)
  styles/        Global CSS
public/          Static assets served as-is
```

## SEO/GEO/AEO

Per-page `<title>`/`<meta description>`/`<link rel="canonical">` are baked into static HTML at build time. JSON-LD schema (Organization, SoftwareApplication, FAQPage) is injected per-page. `/robots.txt` explicitly allows AI crawlers (GPTBot, ClaudeBot, PerplexityBot, Google-Extended). `/sitemap.xml` is auto-generated by `@astrojs/sitemap`.

Source audit: see `WMD Docs/seo-audit-deliverwms-com-2026-05-10.docx`.
